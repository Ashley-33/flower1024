# 🌸 拾光花园 —— 可玩原型 + 可调参配置

> 在线试玩：https://ashley-33.github.io/flower1024/

经典 1024（2048）玩法 + 能量系统 + 5 个道具 + 连击 + 挑战关卡的可玩 H5 原型，
披上「种花·合成·拾阳光」的治愈花园外衣。
纯原生 HTML5 Canvas，**零依赖、零打包**，桌面浏览器 / 手机浏览器 / 微信 webview 均可运行。

## 怎么玩

- **双击 `index.html`** 直接在浏览器里玩（无需起服务）。
- 手机上：把整个文件夹拷过去用浏览器打开，或挂到任意静态服务器 / 微信小游戏 webview。
- 操作：**滑动 / 方向键(WASD)** 移动；**点道具 → 点棋盘选目标**；再点同一道具可取消。
- 当前为**无尽模式**：种花合成、攒阳光、用道具续命，花园长满且无法自救即结束。
  （`config.js` 里仍保留 `levels[]` 作为挑战关设计参考，未接入本 build；game.js 的关卡判定逻辑保留，便于日后开启。）

## 文件结构

| 文件 | 作用 |
|---|---|
| `index.html` | 入口，引入 config.js + game.js |
| `game.js` | 全部游戏逻辑 + Canvas 渲染 + 触控/键盘输入 |
| `style.css` | 移动优先的竖屏样式 |
| `config.js` | **★ 数值真源**：改这里，刷新即生效（游戏运行时读取） |
| `config.json` | config.js 的纯 JSON 镜像，供工具/导入用 |
| `config.csv` | 扁平化数值表，可直接用 Excel 打开调参 |

## 怎么调参（给策划/数值）

只改 **`config.js`**，保存后刷新页面即可，不用任何构建：

- 能量经济 → `energy.mergeGain` / `milestoneBonus` / `combo` / `cap`
- 道具平衡 → `items.*`（费用、冷却、各种枷锁）
- 挑战关卡 → `levels[]`（`target` 目标数字 + `constraints` 约束）

`constraints` 支持的字段：
- `maxSteps`：步数上限
- `minEmptyAtWin`：达成目标时需保留的空格数
- `noItems`：本关禁用全部道具
- `minCumEnergy`：累计获得能量需达到的值

> `config.csv` / `config.json` 与 `config.js` 内容一致，改完记得三者同步（或只把 config.js 当真源，另两个作导出视图）。

## 已实现的设计要点

- **能量经济**：4 合成不给能量（去噪）；高级递增但放缓；首达里程碑一次性奖励；上限 50。
- **道具枷锁**（防止破坏 1024 乐趣）：
  - 时光倒流：撤销会退还该步能量 + 成本逐次翻倍，杜绝“无限回档”。
  - 护盾：原型简化为「3 回合不可移动的墙」，保护大数字。
  - 魔术手：不能移动最大数字 + 6 步冷却，避免沦为“解题器”。
  - 龙卷风：仅当空格 ≤1（真残局）可用。
  - 肥料：只能升级 ≤ 最大数字 1/4 的格、每局限 1 次，且不发里程碑能量。
- **连击**：连续合成回合越长，能量奖励越高。
- **挑战关**：达标判定 + 约束校验 + 胜负结算。

## 倍速模式

棋盘下方有「倍速」选择条：**自动 / 1x / 2x / 3x / 4x**，**按步数解锁**。
- 倍速 m 决定**新方块的起始数字**（基数 = 2^m）：1x→2/4、2x→4/8、3x→8/16、4x→16/32，已有的花不变。
- **解锁规则**：未达步数的档位锁住（显示 `🔒` + 所需步数，不可点）；`config.js` 的 `speed.autoSteps` 默认每 66 步解一档（66/132/198 → 2x/3x/4x）。提示区会显示「再 N 步解锁 Nx」。
- **自动**：跟随已解锁的最高倍速；也可手动点选任一**已解锁**档位（想小数字慢慢玩就选低档）。

## 道具提示

选中任意道具后，道具栏上方会显示该道具的**完整使用说明**（如「🏡 温室罩：点一朵花，它将 3 回合内固定不动、不被挤走」）；即时道具（时光倒流 / 清风）使用后也会回显效果。

## 排行榜（街机风）

- **得分**：经典 2048 算法 = 所有合成结果数值之和，HUD 顶栏 `🏆 得分` 实时显示。
- **卡住时**：弹出「🪴 卡住了！」，提供可用的救场道具（🍃清风 / ⏳时光倒流）或「🏁 结束本局·上榜」。
- **结束时**：弹出榜单弹窗，进了前十会让你**输入名字**留名，高亮自己那一行；只展示**前十**，本地 `localStorage` 持久化。
- **名字校验**：非空、≤8 字、字符集限制（中英数字+少量符号），并过滤违禁词（脏话/政治敏感等）。
  - 违禁词表在 `config.js` 的 `badwords`（已小写去空格归一化匹配）。⚠️ 这是前端兜底示例，**正式上线必须接入服务端内容安全 / 维护中的词库 + 后端校验**——前端名单无法穷举也容易绕过。
- 调参：`config.js` 的 `leaderboard.size`（展示名次数）、`leaderboard.nameMaxLen`（名字长度）。

> `index.html` 的 `?v=` 是开发期的缓存破除参数，改完代码 bump 一下即可强制刷新（本地双击 file:// 打开不受影响）。

## 视觉 / 听觉表现层

- **滑动 + 开花补间**：移动时方块平滑滑向目标（ease-out），合成的花「弹一下」，新方块「破土生长」（easeOutBack）。
- **嫩芽破土造型**：2 / 4 是土堆 + 嫩叶 + 茎的小芽（4 多一颗花苞），8 起才是花，强化低级阶段层次。
- **大花高光**：512+ 的花带柔和**光晕脉动 + 飘落花瓣**粒子，强化高光时刻。
- **传说花（4096+）**：2048 之后继续往上 4096 / 8192 / 16384 / 32768 / 65536，每级一种新配色，并升级为**双层重瓣 + 金色描边 + 环绕星点**；合成阳光奖励继续 +2 递增、里程碑更厚，给高玩持续目标。HUD 实时显示「🌼最高」。
- **声音**（Web Audio 纯合成，无音频文件）：柔和五声音阶环境 BGM；合成音随数值升高；512+ 开花有琶音 sparkle；右上角 🔊 一键静音。
  （受浏览器自动播放策略限制，音频在**首次操作后**才启动。）

## 与设计文档的差异（诚实标注）

1. **关卡数值已调小**便于几分钟内验证手感。原稿“20 步内达到 512”在 2048 机制下数学上不可能（512 需上百次合成），原型改为可达成的速通示意，正式版按真实曲线重设。（本 build 仅无尽模式，关卡逻辑保留未接入。）
2. **护盾**简化为“不可移动的墙”（不实现“仍可被同数字撞入合并”），保留“保护大数字”的核心价值，正式版可再扩展。
